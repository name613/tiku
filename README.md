# 📚 题库格式化助手

一个纯前端的题库格式转换工具，可将任意格式的题目文本（或 Word 文档）一键整理为符合「标准题库导入格式」的规范内容。  
适用于使用题库系统（如教学平台、考试系统）时需要大批量、快速整理题目的场景。

> 🚀 在线使用：部署到 GitHub Pages 后即可直接访问，无需后端服务器。

---

## ✨ 功能特点

- 🔄 **一键格式化**：调用大模型 API（默认 DeepSeek），将混乱的题目自动转为标准格式
- 📎 **支持上传 Word (.docx)**：自动提取文档文本，无需手动复制粘贴
- 📋 **一键复制 / 下载 Word**：格式化结果可直接复制或导出为规范的 .docx 文件
- 🔐 **API Key 本地存储**：Key 仅保存在你的浏览器中，不上传任何服务器，安全可靠
- 🕘 **历史记录**：自动保存最近 20 次转换结果，方便回溯与复用
- 📊 **字符数提示**：输入超过 15000 字符时软提示，避免内容过多影响质量
- 🎨 **响应式设计**：支持电脑、平板、手机等不同设备

## 🛠️ 快速使用

### 在线版（推荐）

将本项目部署到 GitHub Pages 后访问即可（见下方部署说明）。

### 本地运行

1. 下载 `index.html` 文件到本地。
2. 直接用浏览器打开该文件（无需安装任何环境）。
3. 在页面中输入你的 API Key（获取方式见下文），开始使用。

---

## 🔑 获取 API Key

本工具默认使用 **DeepSeek** 大模型 API，性能稳定且价格实惠。

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/)，注册/登录。
2. 进入 **API Keys** 页面，创建一个新的 Key。
3. 复制生成的 Key（`sk-` 开头），粘贴到工具的“API Key 配置”输入框中，点击“保存”。

> 💰 DeepSeek 模型调用费用很低，首次注册通常会有赠送额度，足够大量测试。

---

## ☁️ 部署到 GitHub Pages

1. **Fork 或新建仓库**：在你的 GitHub 中创建一个新仓库（如 `quiz-formatter`）。
2. **上传文件**：将 `index.html` 文件上传到仓库根目录。
3. **开启 Pages**：
   - 进入仓库的 `Settings` → `Pages`
   - `Source` 选择 `Deploy from a branch`
   - `Branch` 选择 `main`（或 `master`），文件夹选 `/ (root)`，点击 Save
4. 等待一两分钟，访问 `https://你的用户名.github.io/仓库名/` 即可使用。

---

## 🧩 技术栈

- 纯前端三件套：**HTML + CSS + JavaScript**
- 文档解析：[mammoth.js](https://github.com/mwilliamson/mammoth.js)（提取 .docx 文本）
- 文档生成：[html-docx-js](https://github.com/evidenceprime/html-docx-js)（生成 .docx 文件）
- 大模型接口：兼容 OpenAI 格式的 Chat Completions API（默认 DeepSeek）

---

## 📄 格式转换说明

本工具内置的《题库格式规范》已涵盖以下常见题型：  
单选题、多选题、判断题、填空客观题、填空主观题、问答题、名词解释、证明/计算/分析题、翻译题、写作题、阅读理解、组合题、听力题等。

所有转换结果严格遵循规范，可直接用于主流题库系统的导入。

---

## ⚠️ 注意事项

- **API Key 只保存在你的浏览器本地**，请妥善保管，不要在公共电脑上勾选“记住”。
- 上传的 Word 文档或粘贴的内容**不会上传到任何服务器**，仅通过 API 发送给大模型进行实时转换。
- 单次输入字符数建议不超过 15000，以获得最佳转换效果。
- 如需切换其他兼容 OpenAI 接口的模型（如硅基流动等），在代码中修改 `API_BASE_URL` 和 `MODEL_NAME` 即可。
